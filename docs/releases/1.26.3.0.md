# sysm 1.26.3.0 — Q3 2026 Security & Release-Cycle Milestone

This release establishes sysm's quarterly release-cycle numbering and consolidates the security hardening delivered in `v1.13.3` with the follow-on secure secret-input work now on `main`.

## Highlights

- Introduces `GENERATION.YY.QUARTER.REVISION` versioning, beginning with the Q3 2026 base release at revision `0`.
- Includes the P2 security remediation batch first shipped in `v1.13.3`, covering updater integrity and stronger plugin, workflow, and process-execution trust boundaries.
- Removes six remaining secret-bearing command-line argument paths for Slack, CalDAV, Keychain, and PDF operations.
- Includes the repaired Podcasts command and playback behavior delivered during the same hardening cycle.

## Breaking security changes

Secrets are no longer accepted directly in process arguments. Affected commands now use a protected terminal prompt, non-TTY standard input, or an explicitly inherited file descriptor. Existing automation that passes Slack tokens, CalDAV app passwords, Keychain values, or PDF passwords as flag values must migrate to the secure input forms documented in the [secure secret-input guide](https://github.com/brndnsvr/sysm/blob/main/docs/guides/secure-secret-input.md).

This intentional compatibility break prevents secrets from being exposed through process listings and shell history.

## Release-cycle policy

`1.26.3.0` means product/UI generation 1, year 2026, quarter 3, base revision 0. Later Q3 releases increment only the final field; the Q4 base becomes `1.26.4.0`.
